# RoleLogic v1.13.0

**Release Date:** May 20, 2026

---

## What's New

### Premium Servers Get a Much Faster Background Sync

Alongside instant real-time processing, RoleLogic runs a periodic background sweep that re-checks every member to catch anything missed in real time — for example during a bot restart or a brief Discord outage. Until now that safety sweep ran at the same pace for every server. Premium servers now get a dramatically faster one.

- **Re-checked about every 2 minutes** instead of every 10.
- **Far more members scanned per pass**, so large servers fully catch up much sooner. A 100,000-member server now fully reconciles in roughly **10 minutes on premium** versus a few hours on free; very large servers (1M+ members) drop from over a day to under two hours.
- **Real-time processing is unchanged and identical on every plan** — when a member's roles change, rules still apply within seconds whether you're free or premium. This release only speeds up how quickly the periodic safety net closes any gap.

Free servers are unchanged: instant real-time processing plus the familiar ~10-minute background sweep — already plenty for the vast majority of servers.

### Premium Servers Now Get Up To 5× More Cross-Server Role Rules

The cross-server role sync limit is now tier-aware. You can keep more role syncs running across multiple Discord servers without juggling which ones to keep.

- **Free plan:** 2 cross-server role rules per server (unchanged).
- **Premium:** up to **10 cross-server role rules** per server.
- **Graceful downgrades.** If your plan ever drops below premium, the extra rules you built up don't get deleted — they're simply paused. The dashboard clearly marks them as locked with the reason ("cross-server limit"), and the moment you re-upgrade or bring your setup back under the free limit, they reactivate automatically.

### A New "Managed Roles" Page

A new **Managed Roles** entry in the sidebar shows you every Discord role that RoleLogic is actively touching on your server — both roles assigned by Role Conditions and roles synced by Role Links.

- See, at a glance, exactly which roles are under automation and which are still purely manual.
- Each entry tells you whether it's managed by a rule, a role link, or both.
- Previously this list lived inside Role Test; it's now a first-class page so you can review your automated roles without running a test.

### Three New Plugin Integrations

RoleLogic can now hand out Discord roles based on three more platforms:

- **Birthday Role.** Auto-grant a celebration role on a member's birthday and remove it the next day — perfect for community-driven birthday shout-outs.
- **osu! Player Role.** Verify a member's osu! account and grant roles based on their global rank, performance points, total play count, supporter status, and more.
- **Bluesky Account Role.** Verify a member's Bluesky account and grant roles based on their follower count, handle, or verified status.

Each integration is browsable directly from the Plugin Marketplace in your dashboard.

### Smarter Conflict Detection Between Rules and Role Links

A Discord role can be managed by either a **Role Condition** or a **Role Link**, but if both try to control the same role, they end up in a permanent add/remove fight. RoleLogic now catches that conflict before it can ever happen.

- **Creating a Role Link** for a role that an existing rule already assigns is now blocked, with a clear explanation telling you which rule already manages that role.
- **Creating or editing a rule** to assign a role that a Role Link already manages is blocked the same way.
- Existing setups are unaffected — this only kicks in when you try to add a brand-new conflict.

The result: no more silent role flapping, no more "the bot keeps removing the role I just added."

### A Clearer Free vs Premium Picture on the Upgrade Page

The Upgrade Plan page now leads with a side-by-side **what you get on Free vs Premium** comparison so you can see the value of upgrading before you ever look at price.

- Categories covered: real-time processing, background safety sync, rule limits, cross-server limits, and more.
- A **NEW** badge highlights the faster premium background sync introduced in this release.
- The comparison sits above the plan cards, so the decision is value first, price second.

### Premium Tracking Built Into the Quota Manager

The Active Quota cards on your dashboard now make it crystal clear when a server is premium and how close it is to becoming premium.

- A **sparkle icon** lights up on every server that has reached the premium threshold.
- A progress bar shows how close the server is to premium — including any quota slots contributed by other co-managers, so the picture is accurate even on shared servers.
- "Remaining slots" guidance tells you exactly how many more slots you'd need to assign for premium to kick in.

### A Brand-New Animated Homepage

The homepage has been redesigned around real, working examples of what RoleLogic can do.

- A cycling **demo rules** carousel shows live examples — "@Booster or @Premium → @VIP", "Twitch follower → @Streamer Fan", "VPN user → no role", and more — with token-by-token reveal animations.
- Each demo lists the integration name, the condition, the action it would take, and a relatable result ("Auto-synced for 12,480 members").
- Updated visuals: subtle gradient hero, refined navigation, and a polished user dropdown.
- All copy is fully translated into every supported language.

### Helpful Banners on the Dashboard Overview

Two new banners on the Dashboard Overview help new and existing users discover what RoleLogic can do:

- A **Role Conditions banner** explains the core concept of role automation with a clear call-to-action to start.
- A **Platforms banner** highlights the integrations available across the plugin marketplace.

### Server Search in the Server Picker

When you have many Discord servers, finding the right one is now much easier — a **search box** at the top of the server picker filters by server name as you type.

### Improved Error Page With One-Click Discord Reporting

When something does go wrong, the new error screen makes it easy to recover and easy to report.

- A **"Report on Discord"** button takes you straight to the support server with the relevant context.
- A **"Show technical details"** toggle hides the noisy stack trace by default — visible only if you (or our support team) need it.
- Cleaner empty states across the dashboard when you have no manageable servers or no role links yet.

## Improvements

### Polished UI Foundation

A long-running refresh of the dashboard's underlying building blocks lands in this release. You'll notice it as a quieter, more consistent feel across every page.

- **Unified buttons, cards, pills, and spinners** so the same kind of element looks the same everywhere.
- **Reusable modal dialogs** with consistent animations, focus handling, and dismissal behaviour.
- A new **accent button** style for high-emphasis actions like "Upgrade" or "Start Now."
- Refined loading skeletons whose dimensions match the final content layout, so pages no longer "jump" as they load.

### Better Accessibility

- **ARIA labels and roles** added across status indicators, navigation, and interactive elements so screen readers can describe them properly.
- A new **live region** announces important state changes — "Rule paused", "Rule scheduled to start", "Saved" — for users who can't see the visual indicator.
- **Higher-contrast text** in several places that were previously borderline for readability.

### Smoother Translations Across the Board

- The **Status page** (`/status`) is now fully translated in all supported languages.
- "Guild" has been replaced with "Server" consistently across Malay and Brazilian Portuguese, matching the rest of the app.
- Many smaller translation tweaks for clarity in role condition previews, conflict messages, and dashboard labels.
- Translations are fully covered in English, Arabic, German, Spanish, French, Malay, and Brazilian Portuguese.

### Quieter, Smarter Notifications

- Error toasts no longer have a close button you can accidentally tap before reading — they auto-dismiss on their own timer instead.
- Notification durations are now tuned per type (success messages dismiss faster than warnings).
- The loading overlay always sits above the rest of the page so it never gets visually buried by a modal underneath.

### Member Origin Role: Clearer Anti-Fraud Story

The Member Origin Role plugin descriptions have been rewritten to better explain its **anti-fraud** capabilities — how it can flag members joining from suspicious sources versus organic ones, and how to use it responsibly.

### Sidebar & Navigation Polish

- A new **server switcher** at the top of the sidebar makes hopping between Discord servers faster, with an invite option built in for adding RoleLogic to a new server.
- The **Linked Guilds** button only shows when you actually have linked guilds, reducing sidebar noise on servers that don't use cross-server rules.
- The "Test All Rules" link is now center-aligned for a more balanced layout.
- Subtle colour and shadow refinements on every sidebar button.

### Dashboard Rule Summaries

A new at-a-glance **rule summary** on the dashboard previews each of your role conditions in plain language ("If member has @Booster, then give @VIP") — translated into every supported language so it reads naturally for your audience.

### Plugin Marketplace Refinements

- A new **Kick Channel Role** icon for that integration.
- The plugin installation panel is more responsive on smaller screens.
- The Role Links empty state now offers a prominent **Browse Plugins** button so first-time users land in the marketplace right away.

## Bug Fixes

### "Bot Not Invited" False Alarms Are Fixed

Some users have reported that after inviting RoleLogic to a Discord server, the dashboard would still show the bot as not yet invited. That false alarm is gone.

- **Your server list refreshes the moment a bot invite completes**, so the freshly invited server appears with the bot marker straight away.
- A **smarter backup check** kicks in if the main bot-presence lookup is briefly uncertain (e.g. during a Discord hiccup or right after a fresh invite). Instead of jumping to "Bot not invited," RoleLogic now asks a per-server question and gets the accurate answer.
- **Faster recovery from short blips.** Uncertain answers now expire from cache in seconds rather than minutes, so any glitch unwinds quickly.

### Smaller Fixes

- **Layout consistency.** Padding and margin across dashboard components have been standardized so cards, banners, and forms line up properly on every screen size.
- **Scrollbar polish.** Consistent scrollbar sizing across panels and modals.
- **Empty state messages.** Clearer wording when you have no servers, no role links, or no rules set up yet.
- **Placeholder text in Role Links** rewritten to make it clearer what URL is expected when adding a custom integration.

## Under the Hood

A round of behind-the-scenes work that powers the user-facing changes above and keeps the dashboard fast and reliable as RoleLogic grows.

- **Tier-aware scheduler.** The background sweep now schedules each server on its own cadence based on plan, instead of one fixed interval for everyone. Premium servers come due more often and are scanned more deeply per pass.
- **Same safety guarantees.** Every Discord request still passes through a global rate-limit budget and a strict concurrency cap, so the faster premium cadence stays comfortably within Discord's limits. Under heavy load the system slows gracefully rather than ever pushing past a safe request rate — premium is faster, never riskier.
- **No configuration needed.** The faster sync and higher cross-server limits apply automatically to servers with active premium quota; they turn off again if premium lapses, with rules locked (not deleted) so a re-upgrade restores them instantly.
- **Same-origin plugin catalogue.** The plugin marketplace's catalogue (plugin list and icons) is now served from the dashboard itself instead of a separate site. The result is a faster-loading marketplace, fewer external dependencies, and quicker rollouts of new plugins.
- **Better SEO and AI-assistant discoverability.** A dedicated **Cross-Server Role Sync** landing page, new static answer pages for common questions ("How do I sync roles between two Discord servers?", "How do conditional roles work?"), updated sitemap, refreshed `robots.txt`, and new `llms.txt` / `llms-full.txt` files so AI assistants can accurately summarise what RoleLogic does.
- **Centralized plugin data on the dashboard.** Plugin information is now loaded once and shared across the dashboard, instead of every page fetching it on its own — cutting redundant requests and making the marketplace snappier.
